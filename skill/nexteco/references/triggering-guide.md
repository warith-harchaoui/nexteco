# Triggering guide

## Should trigger

- estimate what this repo costs to run
- add cost_of_running.yaml
- create a cost-of-running report
- compare local inference versus API cost
- make this repository's runtime cost more explicit
- explain money, time, energy, and carbon for this codebase

## Should usually not trigger

- unrelated general coding help
- purely stylistic README edits
- generic sustainability discussion without repository analysis

## Over-triggering risk

Avoid acting like a generic performance tuning skill or a generic carbon calculator.
Stay focused on repository-native cost-of-running modeling.
