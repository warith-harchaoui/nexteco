# Roadmap

## Near term

- improve report rendering polish
- strengthen validation ergonomics
- add richer examples by repo archetype
- refine the embedded skill
- keep the project attractive to strong OSS candidates
- lightweight CI for tests, example validation, skill packaging, and benchmark smoke checks
- cautious CD for release artifacts once the project surface is stable

## Mid term

- optional JSON export
- optional HTML report rendering
- clearer polyglot wrappers around the Python reference CLI
- better scenario comparison support

## Long term

- stronger ecosystem integrations where justified
- richer public examples from real repositories
- a stable public contract between the OSS repo and the embedded skill
